근데 왜 리터럴 문자열만 String으로 선언하는지? 위에는 char이라고 햇잔아

printf 쓸때 
%s -> string
%d -> double
%f -> float

이거는 알겟는데 맨 뒤에 %n을 붙이는데 이건 모야? 


```java
public class PrintfExample {
    public static void main(String[] args) {
        String name = "김철수";
        int age = 20;
        double score = 95.5;
        
        // printf: 형식 지정 출력 (줄바꿈 없음)
        System.out.printf("이름: %s%n", name);        // %s: 문자열
        System.out.printf("나이: %d세%n", age);       // %d: 정수
        System.out.printf("점수: %.1f점%n", score);   // %.1f: 소수점 1자리
        System.out.printf("합격률: %d%%%n", 85);      // %%: % 출력
        
        // 여러 값 동시 출력
        System.out.printf("%s님은 %d세이고, 점수는 %.2f입니다.%n", 
                          name, age, score);
    }
}
```

여러 변수를 쓸 수도 있고 불러온 변수 뒤에 문자를 그대로 쓸 수도 잇다. % 같은 특수문자를 쓸 때도 그냥 붙이면됨. 
큰따옴표랑 작은따옴표 사이에 차이가 있나 ? 

| 지정자 | 설명 | 예시 |
|--------|------|------|
| `%d` | 10진수 정수 | `123` |
| `%f` | 실수 (기본 소수점 6자리) | `3.140000` |
| `%.2f` | 실수 (소수점 2자리) | `3.14` |
| `%s` | 문자열 | `"Hello"` |
| `%c` | 문자 | `'A'` |
| `%n` | 줄바꿈 | |
| `%5d` | 5자리 확보 (우측 정렬) | `  123` |
| `%-5d` | 5자리 확보 (좌측 정렬) | `123  ` |



3.4 scanner로 입력받기

동일한 프로젝트에서 여러 소스 코드를 당연히 넣을 수 잇다 다른 역할을 하더라도.
처음에 프로젝트와 동일한 이름의 클래스를 만들라고 햇던거는 내가 잘 못 이해한거고
파일명과  public 클래스명이 대소문자까지 일치해야한다는 규칙이다. 

왜 이런 규칙이 잇을까? 
이유 1. 클래스 찾기를 쉽게 하려고

자바 컴파일러나 자바 가상머신이 클래스를 로드할때, 클래스 이름으로 파일을 찾는다. 파일명과 클래스명이 다르면 모든 파일을 각각 열어봐야해 비효율적이다. -> 그러면 이름이 다르면 파일을 못찾나? -> 못찾는게 아니라 컴파일러가 컴파일하지 않는다. 규칙 위반이라서!

이유 2. 개발자도 찾기 쉽게 하려고

이유 3. 하나의 파일 = 하나의 책임
파일 하나에 public 클래스 하나만 허용해서 코드를 명확하게 분리하도록 유도한 철학이다.
따라서  public이 아닌 클래스는 파일명과 클래스명이 달라도 된다.

3.5 형변환

type casting

자동 형변환은 작은 타입에서 큰 타입으로 따로 변경하지 않고 변수 선언만 해도 된다
큰 타입에서 작은 타입으로의 변환은 강제로 해줘야 한다. 

double -> int로의 변환은 바로 되는데 
왜 double을 String으로 변환하는건 에러가 날까? 
```
incompatible types: double cannot be converted to java.lang.String
```
type casting은 같은 계열의 기본형끼리만 가능하다. 
int, float, double, bytes는 모두 동일한 숫자 기본형이다. 
String은 문자 참조형이므로 type casting으로 변환할 수 없고 형 변환 메서드를 사용해야한다.
