4.1 if 문

로그인 시스템 만들기
아이디 비밀번호를 입력하면 맞으면 로그인에 성공합니다. 

처음 시도
import java.util.Scanner;

public class LoginIfs {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        String correctId = "seowoo";
        String correctPassword = "super";

        System.out.println("id:");
        String id = sc.nextLine();
        if (id == correctId) {
          System.out.println("password:");
          String pw = sc.nextLine();
          if (pw == correctPassword) {
              System.out.println("로그인에 성공하였습니다.");
          } else {
              System.out.println("비밀번호가 틀렸씁니다.");
          }
        } else {
            System.out.println("아이디가 존재하지 않습니다.");
        };
    }
}

근데 여기서 맞는 아이디를 입력해도 아이디가 존재하지 않는다고 함. 왜지 ? ?

** `==`는 문자열을 비교하지 않음!!
자바에서 String은 객체, 즉 참조형으로, 이걸로 비교하면 내용이 아니라 같은 객체인지를 비교한다. 
correctId와 id는 다른 객체로 선언되었으므로 두개가 같은지를 비교하면 다르다고 하는 것. 
문자열을 비교하기 위해서는 메서드를 사용해야한다. 

4.2 switch 문

여러 값 중 하나를 선택하도록 할 때
계산기 만들기

switch 문의 다양한 방식을 사용해보려고 이렇게 만들었다. 

```
import java.util.Scanner;

public class CalculatorSwitch {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        boolean valid;
        double result;

        System.out.println("첫번째 숫자를 입력하세요");
        int num1 = sc.nextInt();

        System.out.println("연산자를 입력하세요");
        String calc = sc.nextLine();

        switch (calc) {
            case "-": case "+": case "/" : case "*":
                System.out.println("두번째 숫자를 입력하세요");
                int num2 = sc.nextInt();
                valid = true;
                break;
            default:
                System.out.println("사칙연산만 가능합니다.");
                valid = false;
                break;
        };

        if (valid) {
            switch(calc) {
                case "-":
                    result = num1 - num2;
                case "+":
                    result = num1 + num2;
                case "/":
                    result = num1 / num2;
                case "*":
                    result = num1 * num2;
            System.out.print(result);
            }
        };
    }
}
```

근데 num2가 정의되지 않았다고 하네. 머지???
 -> case 안에서 선언한 변수는 밖에서 못쓴다. 선언과 초기화는 따로 할 수 있는데, 컴파일러가 코드를 빌드하기 전에 체크하면서 사용하기 전에 초기화되지 않을 수 있는 변수에 대해서는 에러를 뱉는다. 이 로직의 경우에는 연산자가 제대로 설정되지 않은 경우에 두번째 숫자도 받지 말라고 햇는데, 그래서 사용하기 전에 초기화가 안되는 경우가 잇을수도 잇다. 그래서 컴파일을 안해주는 것. 
** 스코프 : 변수가 살아있는 범위. 중괄호가 기준이 된다. 
안쪽에서는 바깥 변수 접근 가능, 바깥에서는 안쪽 변수 접근 불가능.

 그리구 scanner에서 nextint나 nextdouble이나로 받은 애들은 엔터가 남아있어서 바로 넘어가버린대.. 
 버퍼를 비워줘야함. 
 그리고 스위치 문에는 break가 있어야 아래 케이스가 쭉 실행되지 않는다.
