# Representational state transfer

인터넷 리소스를 url로 표현하고, http 메서드로 조작하는 아키텍처 스타일

원칙

1. 클라이언트와 서버 분리
2. 서버가 상태 저장 안함
3. 캐싱 가능해야함
4. 일관된 인터페이스
5. 계층화된 시스템
6. 서버가 코드 전송 가능

7. url은 명사로 처리해야함
8. http 메서드로 동작 표현
9. 적절한 http 상태코드를 사용할것
10. stateless
    서버는 클라이언트 상태를 저장하지 않음
11. 필터링, 정렬, 페이징 등 옵션은 쿼리 파라미터로
