jekyll 에서 scss적용하기
메인으로 assete/css/style.scss파일이 스타일시트 역할을 한다. 따라서 여기에 모든 내용을 붙여 넣어서 작성해도 되고, 가독성이나 유지보수 등을 위해서 \_sass 폴더에 부분 파일을 넣고 style.scss 파일에서 임포트해서 쓰는 게 일반적이다.

예를 들어
홈페이지에 섹션을 둔다고 했을때, 이런식으로 만들고 싶다.

--
< 홈페이지 >
-----헤더------
사이드바 섹션1
섹션2
--

그럴때 메인 파일에서 스타일을 import 한다.

> style.scss

```
---
---

@import header
@import sidebar
@import section1
@import section2
@import main
@import layout
@import variables


```

그리고 만들 페이지의 레이아웃은 홈페이지에 적용할 것이므로 홈페이지용 레이아웃을 만든다.

> \_layouts/home.html

```
---
layout: default
---

<div class="home-sections">
  <!-- 섹션1 -->
  <section class="section section-1">
    <h2 class="section-title">섹션1</h2>

      <p>첫 번째 섹션의 내용입니다. 그라데이션 배경과 흰색 텍스트를 사용합니다.</p>
    </div>
  </section>

  <!-- 섹션2 -->
  <section class="section section-2">
    <h2 class="section-title">섹션2</h2>
    <div class="section-content">
      <div class="content-item">
        <h3>서브 제목 1</h3>
        <p>두 번째 섹션의 첫 번째 아이템입니다.</p>
      </div>

        <p>두 번째 섹션의 두 번째 아이템입니다.</p>
      </div>
    </div>
</section>
```

레이아웃 파일은 템플릿 엔진 역할을 한다.

- 템플릿 엔진이란?
  마크다운파일이나 html 파일이 실제로는 이 레이아웃을 감싸서 완성된 웹페이지가 된다.
  또한 같은 구조의 페이지들이 많을 때, 각각의 마크다운 파일에 html 구조를 반복해서 작성할 필요가 없다는 장점이 있다.

예를 들어서 이런 마크다운 파일을 레이아웃으로 감싸서 배포하고 싶은 경우가 있다.
프론트 매터는 이렇게 작성되어 있다.

```
---
layout: post
...
---
```

이 파일은 layouts의 post.html파일을 기본으로 참조하게 된다.
그리고 마크다운의 내용을 {{content}}에서 참조해서 완성된 html 파일을 생성한다.
예를들어서 레이아웃 파일이 이렇게 구성되어있는 경우에

```
---
layout: default
---

<article class="post">
  <header class="post-header">
    <h1 class="post-title">{{ page.title }}</h1>
    <p class="post-meta">{{ page.date | date: "%Y-%m-%d" }}</p>
  </header>

  <div class="post-content">
    {{ content }}  <!-- 여기에 마크다운 내용이 들어감 -->
  </div>

  <footer class="post-footer">
    <div class="post-categories">
      {% for category in page.categories %}
        <span class="category">{{ category }}</span>
      {% endfor %}
    </div>
  </footer>
</article>
```

page의 프론트 매터에서 만든 요소들이 `{ {...} }`안에 들어간다

> `{ {...} }`의 이름은?? 이게 갤러맷인가 그런건가

레이아웃 없이 사용할 수도 있고, 다른 레이아웃을 사용할 수도 있다.
그리고 레이아웃간의 상속관계가 존재해서 상속을 받은 스타일대로 적용될 수도 있다.

여기에 적용되는 스타일은 이렇게 정의한다.

```

.home-sections {
  width: 100%;
  max-width: 1200px;
  margin: 0 auto;
  padding: 0 20px;
}

// 섹션 공통 스타일
.section {
  margin: 60px 0;
  padding: 40px 0;

  &-title {
    font-size: 2.5rem;
    font-weight: 700;
    text-align: center;
    margin-bottom: 40px;
    color: #333;
  }

  &-content {
    line-height: 1.6;
  }
}

// 섹션1 스타일
.section-1 {
  background: linear-gradient(135deg, #667eea 0%, #764ba2 100%);
  color: white;

  .section-title {
    color: white;
    text-shadow: 2px 2px 4px rgba(0,0,0,0.3);
  }

  .section-content {
    font-size: 1.1rem;
    text-align: center;
  }
}

// 섹션2 스타일
.section-2 {
  background-color: #f8f9fa;

  .section-title {
    color: #2c3e50;
    border-bottom: 3px solid #3498db;
    padding-bottom: 10px;
  }

  .section-content {
    display: grid;
    grid-template-columns: repeat(auto-fit, minmax(300px, 1fr));
    gap: 30px;
    margin-top: 30px;
  }
}
```

스타일에도 적용 우선순위가 있다.

1. 전역 스타일 > layout.scss가 정의한다.
2. 레이아웃별 스타일 > 각각의 scss가 정의한다.
3. 페이지별 개별 스타일
   페이지 안에서 개별적으로 스타일을 적용해서 사용할 수도 잇다.

그러니까 스타일은 레이아웃과 독립적으로 있으며, 레이아웃은 html 구조만 담당한다. 스타일은 css 파일에서 관리하며, 모든 페이지가 같은 css 파일을 공유한다.

````
<!-- HTML에서 클래스로 스타일 구분 -->
<div class="post">     <!-- 포스트 스타일 -->
<div class="page">     <!-- 페이지 스타일 -->
<div class="home">     <!-- 홈페이지 스타일 -->
```s
````
