블로그 구성하면서 페이지 로드시 동적으로 어떤일이 일어나는지 디버깅하기가 어려워서 그냥 공부하는 글.
지금 어려움을 겪는건 이런 기능에서다.
기능:
대분류 카테고리 사이드바에서 선택 -> 해당 카테고리의 소분류가 메인 콘텐츠 영역에 카드형식으로 뜸
이 카드를 클릭하면 -> 소분류에 해당되는 포스트가 피드형식으로 뜸

🔍 동작 단계별 분석

1단계: 페이지 로드 시 HTML 생성 (Jekyll/Liquid)

위치: categories.html 라인 120-230

```html
<!-- 1. 카테고리별 서브카테고리 카드 생성 -->

{% for category in sorted_categories %} {% assign subcategories = category_posts
| group_by: "subcategory" %} {% for subcat_group in valid_subcategories %}
<!-- 서브카테고리 카드 -->
<div
  class="subcategory-card"
  data-subcategory="{{ subcat_slug }}"
  data-parent="{{ category_slug }}"
>
  {% endfor %} {% endfor %}

  <!-- 2. 서브카테고리별 포스트 컨테이너 생성 -->

  {% for subcat_group in valid_subcategories %}
  <div
    class="posts-container subcategory-posts"
    data-subcategory="{{ subcat_slug }}"
    data-parent="{{ category_slug }}"
    style="display: none;"
  >
    <!-- 실제 포스트들 -->
  </div>
  {% endfor %}
</div>
```

여기서 생성되는 것들:

- 서브카테고리 카드 (클릭할 수 있는 버튼)
- 서브카테고리별 포스트 컨테이너 (숨겨진 상태)

여기서 사용하는 subcat_slug, category_slug는

- category_slug: 라인 125에서 `{% raw %}{% assign category_slug = category_name %}{% endraw %}`  
- subcat_slug: 라인 143에서 `{% raw %}{% assign subcat_slug = subcat_name %}{% endraw %}`

이 변수들은 Jekyll의 Liquid 템플릿에서 `{% raw %}{% assign %}{% endraw %}` 태그로 선언합니다.

2단계: JavaScript 이벤트 리스너 등록

위치: categories.html 라인 302-450

```javascript
document.addEventListener('DOMContentLoaded', function() {
// 필요한 DOM 요소들 선택
const categoryItems = document.querySelectorAll('.category-item');
const postsContainers = document.querySelectorAll('.posts-container');
const currentCategoryTitle = document.querySelector('.current-category');
```

dom 이란?
**DOM (Document Object Model)**은 웹페이지의 HTML 구조를 JavaScript에서 조작할 수 있게 해주는 인터페이스예요.

  <!-- 이 HTML이 있으면 -->
  <div class="subcategory-card" data-subcategory="genai-llm">
      <h3 class="subcat-title">GenAI&LLM</h3>
  </div>

// JavaScript에서 이렇게 접근할 수 있어요
const card = document.querySelector('.subcategory-card');
console.log(card.dataset.subcategory); // "genai-llm"
console.log(card.querySelector('.subcat-title').textContent); // "GenAI&LLM"

등록되는 이벤트들:

1. 대분류 카테고리 클릭 이벤트
2. 서브카테고리 카드 클릭 이벤트
3. 브레드크럼 네비게이션 이벤트

3단계: 대분류 카테고리 클릭 시 동작

위치: categories.html 라인 307-338

```javascript
categoryItems.forEach((item) => {
  item.addEventListener("click", function () {
    const category = this.dataset.category; // 예: "ML&AI"

    // 1. 사이드바에서 활성 표시
    categoryItems.forEach((cat) => cat.classList.remove("active"));
    this.classList.add("active");

    // 2. 모든 포스트 컨테이너 숨기기
    postsContainers.forEach((container) => {
      container.style.display = "none";
    });

    // 3. 해당 카테고리의 서브카테고리 그리드 표시
    const targetContainer = document.querySelector(
      `[data-category-content="${category}"]`
    );
    targetContainer.style.display = "block";
  });
});
```

확인할 수 있는 것들:

- data-category-content="ML&AI" 컨테이너가 표시되는가?
- 서브카테고리 카드들이 보이는가?

4단계: 서브카테고리 카드 클릭 시 동작 (핵심!)

위치: categories.html 라인 333-353

```javascript
document.addEventListener("click", function (e) {
  if (e.target.closest(".subcategory-card")) {
    const subcatCard = e.target.closest(".subcategory-card");
    const subcategory = subcatCard.dataset.subcategory; // 예: "genai-llm"
    const parentCategory = subcatCard.dataset.parent; // 예: "ML&AI"

    // 1. 모든 컨테이너 숨기기
    postsContainers.forEach((container) => {
      container.style.display = "none";
    });

    // 2. 해당 서브카테고리 컨테이너 찾기 및 표시
    const subcatContainer = document.querySelector(
      `[data-subcategory="${subcategory}"][data-parent="${parentCategory}"]`
    );

    if (subcatContainer) {
      subcatContainer.style.display = "block";
    }

    // 3. 제목 업데이트
    const subcatName = subcatCard.querySelector(".subcat-title").textContent;
    currentCategoryTitle.textContent = subcatName + " 포스트";
  }
});
```

🐛 디버깅 체크포인트

1. HTML 구조 확인

브라우저 개발자도구에서 확인해야 할 것들:

  <!-- 서브카테고리 카드가 올바른 data 속성을 가지고 있는가? -->
  <div class="subcategory-card" 
       data-subcategory="genai-llm" 
       data-parent="ML&AI">

  <!-- 해당하는 포스트 컨테이너가 존재하는가? -->
  <div class="posts-container subcategory-posts" 
       data-subcategory="genai-llm" 
       data-parent="ML&AI" 
       style="display: none;">

2. JavaScript 디버깅 포인트

콘솔에서 확인할 수 있는 것들:

// 1. 서브카테고리 카드들이 올바른 데이터를 가지고 있는가?
document.querySelectorAll('.subcategory-card').forEach(card => {
console.log('Card:', card.dataset.subcategory, card.dataset.parent);
});

// 2. 서브카테고리 포스트 컨테이너들이 존재하는가?
document.querySelectorAll('[data-subcategory]').forEach(container => {
console.log('Container:', container.dataset.subcategory, container.dataset.parent);
});

// 3. 특정 선택자로 컨테이너를 찾을 수 있는가?
const container = document.querySelector('[data-subcategory="genai-llm"][data-parent="ML&AI"]');
console.log('Found container:', container);

3. 가능한 문제점들

1. 데이터 속성 불일치

   - 카드의 data-parent="ML&AI"
   - 컨테이너의 data-parent="ML&amp;AI" (HTML 인코딩)

1. 중복 컨테이너

   - 같은 data-subcategory를 가진 컨테이너가 여러 개

1. CSS 선택자 문제

   - 특수문자(&) 때문에 선택자가 제대로 동작하지 않음

1. 포스트 필터링 문제

   - Jekyll에서 포스트가 올바른 서브카테고리로 분류되지 않음
