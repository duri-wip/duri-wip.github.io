# 이 책에서는 이런 순서로 만들었다

1. 유스케이스 : 유저를 생성한다.
   > main.py : 서버 선언
   > domain/user.py : 도메인 user 객체에서 사용할 도메인 모델을 먼저 작성.
   > domain/user_repo.py : user 도메인을 영속화하기 위한 모듈을 작성한다.
   > ? 영속화란?
   > application/user_service.py : 유저를 생성하고 저장하는 실제 서비스 객체 구현
   > domain/user_repo.py : 도메인 영속화를 위한 모듈에 추상화 메서드 생성
   > infra/user_repo.py : 인프라 계층의 레포지토리 함수를 구현. 도메인 계층의 모듈을 상속한다.
2. 유스케이스 : 중복 유저를 검사하고 중복이 없으면 유저 생성
   > application/user_service.py: 중복 유저를 검사하고 생성하는 로직 구현-find_by_email
   > domain/user_repo.py : 영속화 도메인에서 관련 메서드 생성-find_by_email
3. 유스케이스 : 패스워드를 암호화해서 저장한다.
   > utils/crypto.py: 암복호화를 위한 모듈 구현
   > application/user_service.py: 유저 생성 시 패스워드 암호화 저장 로직 구현
4. 유스케이스 : post 요청을 통해 유저 생성을 할 수 있도록 엔드포인트 구현
   > interface/model.py: 유저 관련 엔드포인트에서 받을 데이터에 대한 모델 생성
   > interface/user_controller.py : 유저 관련 엔드포인트 라우터 생성
   > main.py: router include
5. 유스케이스 : 422 에러를 400 에러로 반환하기
   > main.py: 에러 헨들러 생성
   > interface/user_controller.py: 애플리케이션 계층의 유저생성 서비스 함수를 호출한다.
